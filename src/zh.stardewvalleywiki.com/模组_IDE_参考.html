开始之前

  • 您应当安装 Visual Studio Community（Windows），MonoDevelop（Linux）或 Visual Studio for Mac（Mac）。
  • 以下是一些您应当熟记的基本术语：
    术语 定义
    IDE 用于编辑、运行和编译代码的程序。IDE 是 "Integrated Development Environment"（集成开发环境）的缩写。在 Windows 上，我们主要使用的 IDE 为 Visual Studio，在 Linux/Mac 上则为 MonoDevelop/Visual Studio。
    DLL 扩展名为 .dll 的文件，其中包含编译后的代码。DLL 是 "Dynamic Link Library"（动态链接库）的缩写。SMAPI 通过读取您的 DLL 文件加载您的模组。
    项目（Project） 在 IDE 中，您所编辑的一系列源码、配置、资源（如图片）的集合。每个项目都会被编译为独立的 DLL。
    解决方案（Solution） 一系列项目的集合，包括某些全局设置。解决方案本身不会被编辑，但它可以提供一些便捷的功能（例如，使不同项目能方便地相互引用）。

创建模组项目

在开始编写模组之前，您需要先创建一个解决方案和一个项目以容纳您的模组。

Windows (Visual Studio 2019/2022)

  1. 打开 Visual Studio 2019/2022。
  2. 点击开始界面的 "Create a new project"。
  3. 选择 "C#" 和 "Library"，然后选择 "Class Library"，再点击 "Next"。请一定不要选择 "Class Library (.NET Framework)"，否则无法正常运行。
  4. 在 "configure your new project" 窗口中执行如下操作：
    1. 输入一个模组名称，此名称应当描述您的模组功能。依照惯例，名称应当使用帕斯卡命名法（例如 "PineapplesEverywhere"）。
    2. 请确认您没有勾选 "Place solution and project in the same directory"。
    3. 点击 "Create" 以创建项目。

  2. 然后，选择 .NET 6.0 并点击 "Create" 以创建项目。在 VS 2022 中，您需要翻到下一页。

设置目标框架

目标框架（Target Framework）是您的代码所使用的 .NET 版本，它会影响您模组所需的 .NET 版本。推荐的目标框架为 .NET 6.0，它也是 SMAPI 当前使用的版本。请根据您的系统参阅下面对应小节（Windows/Mac）。

Windows Visual Studio 2019/2022

注意：Visual Studio 2017 可能不再兼容创建模组！如果您在 Visual Studio 2017 中复制-粘贴了模组:制作指南/入门的代码（尤其是 Entry(IModhelper helper) 方法 中的 helper.Events 以及 ButtonPressedEventArgs）），且发现代码报错，请尝试在 Visual Studio 2019/2022 中重新创建项目并复制-粘贴相应的代码。

  1. 打开 Solution Explorer 面板。如果此面板不可见，点击菜单中的 "View > Solution Explorer"：
  2. 在 Solution Explorer 中，右击项目，然后选择 "Properties"：
  3. 在 "Application" 选项卡中，将 "Target Framework" 下拉框设为 ".NET 6.0"：
  4. 此时可能会弹出一个对话框要求您确认更改。点击 "Yes" 以确认：

Visual Studio 2019 for Mac

如果您依照上文的指南搭建了您的项目，则此时框架应当已被设为 .NET 6.0。您如果在前文的步骤中不慎选择了错误的类库，或单纯希望再次检查，则可依照下文的步骤以检查 csproj 文件——这相当于手动设置框架。（如果您单纯希望检查配置是否正确，则无需整体替换 .csproj 文件，而只需检查其中 <TargetFramework> 字段是否为 net6.0 即可。）

很遗憾，我们无法通过 Visual Studio 的界面来更改目标框架。尽管如此，我们仍可以依照 1.5.5 迁移指南中的步骤便捷地更改框架：

  1. 打开待编辑的 .csproj 文件：
  2. 将您模组的 .csproj 文件替换为如下内容（其中 EXAMLE_MOD_NAME 是您的解决方案名称）：
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <AssemblyName>EXAMPLE_MOD_NAME</AssemblyName>
    <RootNamespace>EXAMPLE_MOD_NAME</RootNamespace>
    <Version>1.0.0</Version>
    <TargetFramework>net6.0</TargetFramework>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Pathoschild.Stardew.ModBuildConfig" Version="4.0.0" />
  </ItemGroup>
</Project>
  1. 如果模组使用了 模组:制作指南/APIs/Harmony，请向 PropertyGroup 中添加一行 <EnableHarmony>true</EnableHarmony>。

编辑项目文件(.csproj)

某些情况下您可能希望直接编辑项目文件（主要用于配置构建步骤）。项目文件是一个 .csproj 文件，可以直接在 IDE 中编辑它。

Visual Studio 2017

  1. 打开 Solution Explorer 面板。如果此面板不可见，点击菜单中的 "View > Solution Explorer"：
  2. 在 Solution Explorer 面板中，右击项目，然后选择 "Unload"：
  3. 再次右击项目，然后选择 "Edit <project name>.csproj"：
  4. 在出现的编辑界面中进行编辑，然后保存。
  5. 完成以上步骤后，再次右击项目，然后选择 "Reload Project"：

MonoDevelop/Visual Studio for Mac

  1. 打开 Solution 面板。如果此面板不可见，点击菜单中的 "View > Pads > Solution"：u:
  2. 在 Solution 面板中，右击项目然后选择 "Tools > Edit File"：
    。
  3. 在出现的编辑界面中进行编辑，然后保存。

引用其他DLL

Visual Studio 2017

  1. 打开 Solution Explorer 面板。如果此面板不可见，点击菜单中的 "View > Solution Explorer"。
  2. 在 Solution Explorer 面板中，右击 "Reference"，然后点击 "Add Reference..."：
  3. 在 "Reference Manager" 窗口中，选择左侧的 "Browse"，然后点击底部的 "Browse..."：
  4. 找到欲引用的 DLL，选中它，然后点击 "Add"：
  5. 在 "Reference Manager" 窗口中，点击 "OK"：

Rider

首次打开项目时，应当添加对 StardewModdingAPI.dll 的引用，以启用代码补全。

  1. 在 Explorer 面板中，选择 "Solution" 视图。
  2. 在项目下，右击 "Dependencies"，然后点击 "Reference..."。
  3. 在 "Add Reference" 窗口中，点击底部的 "Add From..."。
  4. 找到欲引用的 DLL，选中它，然后点击 "OK"。
