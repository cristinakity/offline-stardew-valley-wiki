Format

The data file is a list of objects which have the following four elements.

  • RequiredTags - A list of object context tags.
  • SpawnTime - An integer
  • ProducedItems - A list of objects describing possible pond production.
  • PopulationGates - A dictionary of objects controlling population limits and quests.

Each element is described in more detail below.

RequiredTags

To find the definition for a particular pond, the game goes through the pond definitions in the order they appear in the data file. The first definition where all of the required tags match the type of fish in the pond will be used. Individual fish are generally referenced with a tag that looks like item_<fishname> (e.g., item_sunfish). These tags are automatically created by the game for each fish object.

There are some more general tags defined in Content\Data\ObjectContextTags.xnb which usually match multiple fish. For example the tag fish_carnivorous is assigned to the Dorado, Lingcod, and Pike. The last pond definition in the unmodded Fish Pond data has a required tag of category_fish and is a generic catch-all that matches any fish in the -4 category.

Although unused in the base game, a tag can be prefaced with an exclamation point to reverse the logic; this will instead match any fish which does not have the associated tag assigned. For example, a definition with a required tag of !fish_carnivorous would match any fish except for Dorado, Lingcod, and Pike.

SpawnTime

This element controls how many days it takes for the pond to produce a new fish, assuming there is still room. The Tiger Trout is set to a spawn time of 999999 days which makes it essentially unable to reproduce, even though technically it would eventually do so if the game was played long enough. If the spawn time is set to -1, the game will dynamically determine the spawn time using the base price of the fish as shown below.

price range spawn time
0 - 30 1 day
31 - 80 2 days
81 - 120 3 days
121 - 250 4 days
251+ 5 days

ProducedItems

This element defines which items will be produced. Note that there is a base item production chance of (0.15 + 0.08 * population of pond) defined in code and this element will only be processed to select the actual item if that initial check is passed. Each entry contains the following 5 fields:

  • RequiredPopulation - integer
  • Chance - float
  • ItemID - integer (object ID)
  • MinQuantity - integer
  • MaxQuantity - integer

The entries are processed in order. For each, the game checks to see if the pond has a population of at least the RequiredPopulation amount. If so it rolls a random number in the range [0,1) and checks to see if that value is less than or equal to the given Chance. If so, the pond will produce between MinQuantity and MaxQuantity of the object specified by ItemID. If not, it will move on to the next element in the list and do the same check. This process repeats until either an item is selected or there are no more entries. Note that if the item is Roe, the game will set the name, color, and value of the Roe based on the type of fish in the pond.

PopulationGates

This element controls the population limits and quest items needed to bypass that limit. Note that all ponds have a final limit of 10 fish which is defined in code. If this element is null, there will be no quests and the starting population will be 1/10. The gates themselves are a dictionary where the key is a population number and the value is an array of possible quest items. When the game detects that the pond should grow to the given population number, it will start the quest and randomly choose one of the possible items for the request.

The possible quest items are strings containing up to 3 integers separated by spaces. These are the object ID (required), minimum quantity (optional), and maximum quantity (optional). If both minimum and maximum are supplied, there will be a random roll in that range to choose the requested quantity. If only the minimum is supplied, that exact amount will be requested. And if neither minimum or maximum is supplied, the requested quantity will be one.

Example: The fish_super_cucumber definition contains the following entry in its population gates: 

      "6": [
        "422 2 3",
        "60 2",
        "749 2 3",
        "116"
      ],

This entry means that in order to grow from population 5 to population 6, a quest will be initiated, and that quest will require one of the following 4 items (randomly picked with equal weight):

  • 2-3 Purple Mushrooms
  • 2 Emeralds
  • 2-3 Omni Geodes
  • 1 Dried Starfish
