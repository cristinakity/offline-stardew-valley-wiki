Hey, there is a difference between the pages Bait/Isca and Bait (item)/Isca (item). And I'm not sure 100% how the name template handles colons, but the warp totems entries aren't identical either. Dubesor (discussão) 21h33min de 27 de abril de 2021 (UTC)

I'll explain how it works: the "name template" takes the name described in {{name|item name}} and puts .png at the end, that is, the name of the image; for example, to write Isca you have to write {{name|Bait}} and not {{name|Bait (item)}} which looks like this 24px Bait (item) because of the name of the image (Bait.png). I did all the modifications and I would like to know if I can also add the parameter class=inline to the NPC Template. I think this can be very useful, as it is in the Name Template. JaksStrange (discussão) 21h22min de 7 de junho de 2021 (UTC)
I'm ok with allowing for the template to display inline, but Template:NPC is not like Template:Name in the way it displays things. Template:Name uses .css styling, while Template:NPC just adds paragraph tags (<p>...</p>). So, the change would consist of replacing
<p>
with something like
{{#ifeq:{{{placement|}}}|inline||<p>}}
and
</p>
with
{{#ifeq:{{{placement|}}}|inline||</p>}}
I'd like to stay away from naming the parameter "display" or "position" or anything else that refers to a standard .css keyword. "Placement" isn't used in official .css as far as I know, so that would be a good choice, although you could name it almost anything reasonable. margotbean (discussão) 19h09min de 8 de junho de 2021 (UTC)
I did some tests and got a result that worked, it was as follows:
use this in the beginning
{{#ifeq:{{{class|}}}|inline|<span class="nametemplateinline">|<p>}}
and this in the end
{{#ifeq:{{{class|}}}|inline|</span>|</p>}}
Can this be used? JaksStrange (discussão) 19h17min de 8 de junho de 2021 (UTC)
Testing what you suggested, I see that just
