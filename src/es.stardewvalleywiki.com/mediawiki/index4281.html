Formato

El archivo contiene dos tipos de datos:

  • Gustos Universales aplican a PNJs en general. Usan la clave Universal_Gusto, y su valor es un conjunto de IDs separadas por espacio (objeto item ID si es ≥0, ID de categoría si es <0). Por ejemplo, considera esta entrada: 
      "Universal_Like": "-2 -7 -26 -75 -80 72 395 613 634 635 636 637 638 724 459"
    Significa que todo PNJ tendría un gusto universal por la categoría -2, el objeto 72, etc.
  • Gustos Personales se aplican a un PNJ específico. La clave es el nombre interno de un PNJ (como Abigail), y su valor alterna texto (o respuesta) e IDs en este orden: amado, gustado, disgustado, odio, y neutral. Por ejemplo, considera los gustos de Abigail: 
      "Abigail": "¡Me encanta, de verdad! ¡Eres la bomba, @!/66 128 220 226 276 611 904/¡Hala! ¿Cómo sabías que me rugía la tripa? ¡Tiene una pinta deliciosa!//¿Qué se supone que voy a hacer con esto?/-5 -75 -79 16 245 246/¿En qué estabas pensando? ¡Es horrible!/330/¿Me has traído un regalo? Gracias.//",

    Al separar el texto con / como delimitador, obtenemos estos datos:

    index gusto diálogo de reacción IDs de referencia
    0, 1 amado ¡Me encanta, de verdad! ¡Eres la bomba, @! 66 128 220 226 276 611
    2, 3 gustado ¡Hala! ¿Cómo sabías que me rugía la tripa? ¡Tiene una pinta deliciosa! none
    4, 5 disgustado ¿Qué se supone que voy a hacer con esto? -5 -75 -79 16 245 246
    6, 7 odiado ¿En qué estabas pensando? ¡Es horrible! 330
    8, 9 neutral ¿Me has traído un regalo? Gracias. none
    Esto significa que, como gusto personal, ama el item 66 (Ametista), le disgusta la categoría -5 (huevos), etc.

Cómo se determina el gusto

Este formato permite resolver problemas entre gustos

  • Entre una item ID y una ID de categoría;
  • Entre gusto universal y personal;
  • Entre valores contradictorios (e.g., Jodi ama y odia los narcisos (item ID 18));
  • Y cualquier combinación de estas (e.g., entre un item universal y una categoria personal).

El juego usa un algoritmo algo complicado para determinar cuánto un PNJ le gusta un regalo (ve NPC::getGiftTasteForThisItem). Aquí hay una versión limpia del algoritmo en pseudo-código:

considera "if" como "si [condición]", y "else if" como "si lo anterior no funciona, revisa [condición]". "return" es el valor que devuelve. 

var GUSTO = neutral
bool TIENE_ID_UNIVERSAL = false
bool TIENE_ID_UNIVERSAL_NEUTRAL = false

// parte I: gusto universal por categoría
if categoría es universalmente amado:
   GUSTO = amor
else if categoría es universalmente odiado:
   GUSTO = odio
else if categoría es universalmente gustado:
   GUSTO = gusto
else if categoría es universalmente disgustado:
   GUSTO = disgusto

// parte II: gusto universal por item ID
if itemID es universalmente amado:
   GUSTO = amor
   TIENE_ID_UNIVERSAL = true
else if itemID es universalmente odiado:
   GUSTO = odio
   TIENE_ID_UNIVERSAL = true
else if itemID es universalmente gustado:
   GUSTO = gusto
   TIENE_ID_UNIVERSAL = true
else if itemID es universalmente disgustado:
   GUSTO = disgusto
   TIENE_ID_UNIVERSAL = true
else if itemID es universalmente neutral:
   GUSTO = neutral
   TIENE_ID_UNIVERSAL = true
   TIENE_ID_UNIVERSAL_NEUTRAL = true

// parte III: anular neutral si es de categoría universal
if GUSTO es neutral y no TIENE_ID_UNIVERSAL_NEUTRAL:
   if item es comestible pero sabe mal (-300 > comestibilidad < 0):
      GUSTO = odio
   else if item vale < 20g:
      GUSTO = disgusto

// parte IV: a veces se anula por gustos personales
if ((npc ama itemID O (item tiene categoría Y npc ama categoría)) Y (item no tiene categoría O npc no tiene amor por categoría O no hay gusto universal para itemID)
   return amor
if ((npc odia itemID O (item tiene categoría Y npc odia categoría)) Y (item no tiene categoría O npc no tiene odio por categoría O no hay gusto universal para itemID)
   return hate
if ((npc le gusta itemID O (item tiene categoría Y npc le gusta categoría)) Y (item no tiene categoría O npc no tiene gusto por categoría O no hay gusto universal para itemID)
   return gusto
if ((npc le disgusta itemID O (item tiene categoría Y npc le disgusta categoría)) Y (item no tiene categoría O npc no tiene disgusto por categoría O no hay gusto universal para itemID)
   return disgusto
if ((npc neutrals itemID O (item tiene categoría Y npc neutrals categoría)) Y (item no tiene categoría O npc no tiene neutral por categoría O no hay gusto universal para itemID)
   return neutral

// parte V: devuelve el gusto si no fue anulado
return GUSTO
